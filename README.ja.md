# 🦞 openclaw-insight

> [OpenClaw](https://github.com/openclaw/openclaw) の使用分析と改善提案ツール — AI アシスタントの利用習慣を分析し、実行可能な最適化を提案します。

[English](./README.md) | [简体中文](./README.zh-CN.md) | **日本語**

[Claude Code の `/insights` コマンド](https://docs.anthropic.com/en/docs/claude-code)にインスパイアされた `openclaw-insight` は、ローカルの OpenClaw セッション履歴を分析し、使用統計、行動パターン、フリクション分析、改善提案を含むインタラクティブなレポートを生成します。

## 機能

### 📊 包括的な使用統計
- **セッション指標**: 合計数、日次平均、アクティブ日数、連続使用記録
- **トークン消費**: 入出力の内訳、キャッシュヒット率、コスト見積もり
- **時間分析**: 日次アクティビティチャート、ピーク時間帯の特定
- **チャンネル分布**: チャンネル別セッション数、トークン効率比較
- **モデル使用状況**: マルチモデル分析、モデル別キャッシュパフォーマンス

### 🔍 行動パターン検出
- **ピーク時間帯** — 最もアクティブな利用時間帯を特定
- **チャンネル選好** — プラットフォーム別の利用分布を分析
- **セッション長プロファイル** — クイッククエリと深い対話を区別
- **キャッシュ利用率** — プロンプトキャッシュの有効性を測定
- **モデル多様性** — マルチモデル利用パターンを追跡
- **ツール選好** — 最も使用頻度の高いアシスタントツールをランキング

### 🔥 フリクション分析
セッション内のペインポイントを自動検出:

| フリクションタイプ | 説明 |
|---|---|
| トークン浪費過多 | 出力/入力比が異常なセッション |
| 過度なコンパクション | コンテキスト制限に繰り返し到達する会話 |
| 放棄されたセッション | 開始されたがほとんど使用されなかったセッション |
| キャッシュ未活用 | 大規模セッションでキャッシュヒットなし |
| コンテキストオーバーフロー | コンテキストウィンドウの繰り返し枯渇 |
| 単一メッセージセッション | 高オーバーヘッドの短時間インタラクション |

### 💡 実行可能な改善提案
影響度と実装難易度で分類された提案:

- **トークン効率** — キャッシュ最適化、冗長性制御、バッチ処理
- **チャンネル最適化** — マルチチャンネルアクセス、チャンネル別効率分析
- **モデル選択** — 簡単なタスクをより経済的なモデルにルーティング
- **コンテキスト管理** — 会話分割、トークンバジェット制御
- **スケジュール最適化** — 利用パターンの最適化
- **メモリ活用** — セッション間コンテキスト保持
- **機能発見** — 活用されていない OpenClaw 機能
- **ワークフロー改善** — 対話深度の最適化、要件の明確化

### 📈 インタラクティブ HTML レポート
美しいダーク/ライトテーマレポート:
- Chart.js 搭載のインタラクティブな可視化
- 展開可能な提案カード（設定コードスニペット付き）
- ソート可能なセッション詳細テーブル
- 全レスポンシブデザイン
- **100% ローカル実行** — データのアップロードなし

## インストール

### ワンクリックインストール（推奨）

```bash
curl -fsSL https://raw.githubusercontent.com/openclaw/openclaw-insight/main/install.sh | bash
```

### npm でインストール

```bash
# グローバルインストール
npm install -g openclaw-insight

# または直接実行
npx openclaw-insight
```

### ソースからビルド

```bash
git clone https://github.com/openclaw/openclaw-insight.git
cd openclaw-insight
npm install
npm run build
node dist/index.js
```

## 使い方

```bash
# 基本的な使い方 — 過去30日を分析、ブラウザでレポートを開く
openclaw-insight

# 過去7日のみ分析
openclaw-insight --days 7

# JSON形式で出力
openclaw-insight --format json --output report.json

# カスタム状態ディレクトリを指定
openclaw-insight --state-dir /path/to/.openclaw

# 特定のエージェントを分析
openclaw-insight --agent my-agent-id

# 詳細モード
openclaw-insight --verbose

# ブラウザを自動的に開かない
openclaw-insight --no-open
```

## コマンドラインオプション

| オプション | デフォルト | 説明 |
|---|---|---|
| `-d, --days <n>` | `30` | 分析日数 |
| `-m, --max-sessions <n>` | `200` | 最大処理セッション数 |
| `-a, --agent <id>` | 自動検出 | 分析するエージェント ID |
| `-s, --state-dir <path>` | `~/.openclaw` | OpenClaw 状態ディレクトリ |
| `-o, --output <path>` | `~/.openclaw/usage-data/report.html` | 出力ファイルパス |
| `-f, --format <fmt>` | `html` | 出力形式（`html` または `json`） |
| `--no-open` | — | レポートを自動的に開かない |
| `-v, --verbose` | — | 詳細出力を有効化 |

## プライバシー

- **100% ローカル** — すべての分析はあなたのマシンで実行
- **アップロードなし** — データがシステムから出ることはありません
- **テレメトリなし** — ツール自体は使用データを一切収集しません
- **コンテンツ非参照** — インタラクションパターンのみに焦点、会話内容は分析しません
